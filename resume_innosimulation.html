PROBLEM 01
라이브러리 소스 직접 분석 — Android 생명주기 레벨 근본 해결 (C++ 코드베이스 분석 능력과 직결)
스크롤 시 오디오 이중 출력 → API 문서로 해결 불가 판단 → 원본 라이브러리 소스 직접 추적 → 생명주기 분리로 완전 해결
딜라이브 마이그레이션에서 쇼츠형 피드 스크롤 시 이전 영상의 오디오가 겹쳐 출력되는 문제 발생. 단순 재생 메서드 재호출로 내부 생명주기를 제어할 수 없다는 사실을 인지하고, 원본 플레이어 라이브러리 소스를 직접 열어 초기화·재생·해제 흐름을 파악한 뒤 생명주기를 명시적으로 분리 호출해 근본 해결. 이 접근 방식은 C++ 시뮬레이터 코드베이스에서 문서화되지 않은 동작 원리를 추적할 때 동일하게 적용됩니다.
기존 접근 (표면 수준)
재생 API 재호출 → 내부 상태 미초기화 → 오디오 잔존
소스 분석 후 (근본 해결)
생성 / 재생 / 해제 단계 명시적 분리 호출 → 오디오 겹침 완전 제거
PROBLEM 02
실시간 동시성 문제 — 서버 중심 상태 관리로 데이터 일관성 확보
다중 클라이언트 동시 입력 → 상태 충돌·불일치 가능성 → 서버 단일 상태 원천 설계 → 충돌 없는 실시간 동기화
실시간 숫자 대결 게임에서 여러 클라이언트가 동시에 값을 변경하면 데이터 불일치가 발생하는 동시성 문제. 서버를 단일 상태 원천(Single Source of Truth)으로 설계해 모든 상태 변경 요청을 서버에서 순차 처리. 클라이언트에는 Optimistic Update로 즉각 UI 반응을 주고 서버 확정값으로 최종 동기화. C++ 시뮬레이터에서 멀티스레드 환경의 mutex·조건 변수를 통한 상태 직렬화와 동일한 사고.
PROBLEM 03
실시간 영상 처리 성능 확보 — 제한된 환경에서 25FPS 실시간 추론
전체 프레임 처리 시 추론 속도 부족 → 경량 YOLOv5 모델 선택 → 프레임 샘플링 + ROI 분석 → 평균 25FPS 이상 확보
CCTV 실시간 이상 감지 시스템에서 일반 컴퓨팅 환경에 전체 프레임을 그대로 추론했을 때 실시간 처리가 불가능했습니다. YOLOv5 경량 모델 채택, 프레임 샘플링(N 프레임당 1회 추론), 관심 영역(ROI) 중심 처리를 조합해 평균 25FPS 이상의 실시간 처리 성능을 확보. OpenCV Mat 구조·BGR 변환·프레임 파이프라인은 C++ 네이티브 구현과 직접 대응됩니다.
PROBLEM 04
오탐률 알고리즘 개선 — 복합 조건 + Temporal 필터로 30% → 10% 이하
단순 객체 탐지로 오탐 30% → 바운딩 박스·중심 좌표·자세 분석 결합 → Threshold + Temporal 조건 적용 → 오탐률 10% 이하 달성
단순 객체 탐지만으로는 기대기·구분리기 등 일상 동작을 이상으로 오인하는 오탐률이 약 30%에 달했습니다. 바운딩 박스 종횡비, 중심 좌표 이동 거리, 자세 유지 시간을 복합 조건으로 결합하고 Threshold 필터링과 Temporal 조건(지속 시간 임계)을 추가해 오탐률을 10% 이하로 감소. 파라미터 기반 조건 튜닝과 성능 측정을 반복하는 방식은 시뮬레이터 파라미터 최적화 작업과 유사한 접근.
PROBLEM 05
멀티 디바이스 상태 분리 설계 — 연결·기기·콘텐츠 단위 독립 관리
3대 셋톱박스 상태 혼재·렌더링 오류 → 도메인별 상태 분리 설계 → 각 상태 독립 변경·구독 구조 완성
헬로tv 마이그레이션에서 셋톱박스 최대 3대의 상태를 단일 구조로 관리하면 상태가 혼재되어 잘못된 콘텐츠가 노출되는 문제 발생. 연결 상태 / 기기 노출 / 콘텐츠 접근 권한을 각각 독립된 관리 단위로 분리해 각 상태가 독립적으로 변경·구독되도록 재설계. 다중 입출력을 처리하는 시뮬레이터에서 각 장치 상태를 독립 모듈로 분리 관리하는 구조와 동일한 설계 원칙.
PROBLEM 06
네트워크 인프라 구조적 해결 — Elastic IP + HTTPS 전 구간 보안 적용
로드밸런싱 환경 IP 변경 → 방화벽 차단 → EC2 Elastic IP 고정 구조 재설계 → SSL 인증서 Proxy 서버 직접 적용 → 안정적 통신 + 전 구간 HTTPS 보안
로드밸런싱 환경에서 Proxy 서버 IP가 변경되며 방화벽에 의해 요청이 차단되는 인프라 문제 발생. EC2 기반 Proxy 서버에 Elastic IP를 할당해 고정 IP 구조로 재설계하고, HTTP→HTTPS 전환 시 SSL 인증서를 Proxy 서버에 직접 적용해 전체 통신 구간을 HTTPS로 안전하게 전환. 시스템 레벨 문제를 구조적으로 접근해 근본 해결한 사례.
