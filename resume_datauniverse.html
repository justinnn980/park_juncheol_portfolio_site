실서비스에서 맞닥뜨린 문제와 해결 과정

PROBLEM 01
쇼츠형 화면 오디오 이중 출력 — 라이브러리 소스 레벨에서 근본 원인 해결
스크롤 시 이전 영상 오디오 잔존 → API 문서로 해결 불가 판단 → 원본 라이브러리 소스 직접 분석 → 오디오 겹침 완전 해결
딜라이브 마이그레이션에서 쇼츠형 피드를 스크롤할 때 이전 영상의 오디오가 남아 겹쳐 출력되는 문제 발생. 단순히 재생 메서드를 재호출하는 방식으로는 플레이어 내부 생명주기를 제어하기 어렵다고 판단. 원본 플레이어 라이브러리 소스를 직접 열어 내부 초기화·재생·해제 흐름을 파악한 뒤, 영상 전환 시 생성 / 재생 / 종료 단계를 명시적으로 분리 호출하도록 수정. 오디오 겹침 문제 완전 해결.
PROBLEM 02
Riverpod으로 셋톱박스 3대 상태 — 연결·기기·콘텐츠 단위 분리 설계
최대 3대 상태 혼재 · 렌더링 오류 → 도메인별 StateNotifierProvider 분리 → 상태 변화 즉시 반영 · 유지보수 명확
헬로tv 마이그레이션에서 셋톱박스 최대 3대 페어링 상태를 단일 Provider로 관리하면 상태가 혼재되어 잘못된 콘텐츠가 노출되는 문제 발생. 연결 상태 / 기기 노출 / 콘텐츠 접근 권한을 각각 별도 StateNotifierProvider로 분리해 각 단위의 상태가 독립적으로 변경·구독되도록 재설계. 이후 신규 기기 추가 기능 붙일 때도 영향 범위가 명확해져 유지보수 속도 향상.
PROBLEM 03
앱 배포 주기 의존 탈피 — WebView 분리로 운영 구조 개선, FCP 39% 단축
화면 변경마다 앱 재배포 필요 → 탐색 화면 React + WebView 분리 → FCP/LCP 지표 기반 최적화 → 즉시 반영 운영 + FCP −39%
콘텐츠 구성·배너·화면 변경마다 앱 재배포가 필요해 운영 비용이 높고, WebView 전환 후 초기 로딩이 느려지는 부작용도 발생. React 기반 웹으로 탐색 화면을 분리하고 Flutter ↔ WebView JavaScript Bridge 양방향 통신 구조를 설계해 앱 배포 없이 즉시 반영 가능한 운영 구조 구현. FCP/LCP 지표로 병목을 찾아 Lazy Loading·스켈레톤 UI·CDN을 적용해 FCP 39% 단축, LCP 10.8% 개선.
변경 전
화면 변경 → 코드 수정 → 앱 빌드 → 스토어 심사(1~3일) → 배포
변경 후
화면 변경 → React 웹 배포 → 즉시 반영 (앱 배포 없음)
PROBLEM 04
NCG DRM → Multi DRM 전환 — 단일 플레이어 인터페이스로 통합
서비스마다 다른 DRM 방식 → DRM 추상화 레이어 설계 → 단일 플레이어 인터페이스로 통합 → 서비스 무관 동일 사용자 경험
딜라이브 앱(NCG DRM)과 헬로tv 앱(Multi DRM: Widevine/FairPlay) DRM 방식이 달라 재생 흐름이 서비스마다 달라지는 문제. 서비스별로 별도 플레이어를 만들면 유지보수 비용이 두 배가 된다고 판단. 기존 플레이어를 커스터마이징해 DRM 방식을 내부에서 추상화하고 사용자에게는 동일한 UI·요청 흐름으로 콘텐츠를 재생할 수 있도록 단일 인터페이스로 통합.
PROBLEM 05
Flutter 2 → 3 메이저 버전업 — 전체 의존성 재검토 후 안정적 완료
Flutter 3 업그레이드 필요 → DRM·플레이어·상태관리 라이브러리 전체 검토 → 비호환 라이브러리 교체 · Null Safety 수정 → 서비스 중단 없이 업그레이드 완료
딜라이브 마이그레이션 중 Flutter 2 기반 코드베이스의 Flutter 3 업그레이드 필요. 단순 버전 태그 변경으로 끝나지 않고 DRM 라이브러리, 미디어 플레이어, Riverpod, 기타 의존 라이브러리의 Flutter 3 호환 여부를 전수 검토. 비호환 라이브러리 교체 버전 확인 및 Null Safety 마이그레이션이 필요한 코드를 직접 수정해 기능 이상 없이 안정적으로 업그레이드 완료.
PROBLEM 06
로드밸런싱 환경의 방화벽 차단 — AWS Elastic IP + HTTPS로 해결
Proxy IP 변경 → 방화벽 차단 → EC2 Proxy에 Elastic IP 고정 → Nginx SSL 인증서 적용 → 전구간 HTTPS 보안 통신 완성
로드밸런싱 환경에서 Proxy 서버 IP가 바뀌면서 이통사 방화벽에 차단되어 앱-서버 간 통신이 끊기는 문제 발생. EC2 기반 Proxy 서버에 Elastic IP를 할당해 고정 IP 구조로 전환하고, Nginx에 Let's Encrypt SSL 인증서를 적용해 HTTP에서 HTTPS로 전환. 방화벽 차단 문제를 해결하면서 동시에 전구간 보안 통신 구조를 완성.
