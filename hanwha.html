⚡
성능 최적화
Redis 캐싱·인덱스 설계로 DB 부하 제거
@Cacheable/@CacheEvict + TTL 60초로 반복 쿼리를 캐싱하고, 복합 인덱스 2개(user_id,id / user_id,completed,id)를 직접 설계해 성능을 최적화했습니다. 대용량 보험 시스템의 서비스 안정화 업무에 바로 기여할 수 있습니다.

보안 설계
Refresh Token Rotation · 404 마스킹 보안 설계

LG헬로비전(가입자 343만 명), 드라이브(매출 3,500억 원) 등 구조가 전혀 다른 대규모 외부 시스템들을 단일 확장 가능한 구조로 연동하는 API를 직접 설계하고 운영했습니다. AES-256 암호화 통신, Quartz 배치 자동화, Elastic IP 기반 인프라 재구성까지 혼자서 완결지은 경험이 있습니다.

퇴직 후에는 Study Tracker API(JWT 직접 구현, Redis 캐싱, 멱등성, CQRS)와 다마(Kotlin, DDD, Refresh Token Rotation) 프로젝트를 통해 금융 시스템 수준의 신뢰성과 보안을 직접 설계·구현하며 역량을 키웠습니다.

한화생명 보험코어 시스템이 추구하는 MSA, 클라우드/컨테이너, 고신뢰성은 제가 실무에서 지향해온 방향과 정확히 일치합니다. 빠른 속도로 변화하는 금융 디지털 환경에 민첩하게 대응하는 팀에 기여하고 싶습니다.

Study Tracker API

2025 · 개인 프로젝트 · Java 21 · Spring Boot · PostgreSQL · Redis · Docker

프로젝트 목적

백엔드 실무 전체 흐름(인증·캐싱·무결성·테스트·배포)을 하나의 프로젝트에 집약해 금융 시스템 수준의 신뢰성을 직접 구현하고 검증했습니다.

핵심 구현 내역

  • JWT 직접 구현 : jjwt 없이 HMAC-SHA256 서명 구현, MessageDigest.isEqual()로 타이밍 공격 방어
  • Redis 캐싱 최적화 : @Cacheable/@CacheEvict + TTL 60초, 복합 인덱스 2개 설계(user_id,id / user_id,completed,id)로 반복 쿼리 부하 제거
