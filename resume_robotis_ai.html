Section 02
데이터/모델 품질을 직접 개선한 사례
CASE 01 · 라벨 정확도 검증 — 핵심 직무 직결
바운딩박스·Confidence Score 정량 분석으로 오탐률 30% → 10% 이하
YOLOv5 추론 결과 오탐/미탐 다수 발생 → 종횡비·중심좌표 이동·자세 유지시간 정량 기준 설계 → Threshold 튜닝 및 복합 조건 적용 → 오탐률 30%→10%↓ · 25FPS+
CCTV 실시간 낙상 감지 시스템에서 YOLOv5 모델의 출력 결과(바운딩 박스, Confidence Score)를 분석해 오탐/미탐 패턴을 정량적으로 분류했습니다. 바운딩 박스 종횡비, 중심 좌표 이동, 자세 유지 시간을 복합 조건으로 묶고 Threshold를 튜닝해 판단 로직과 데이터 품질을 구조적으로 개선했습니다. 이는 로보티즈AI가 요구하는 라벨 정확도·일관성 검증, 데이터셋 품질 분석 업무와 본질적으로 같은 작업입니다.
개선 전
단순 Confidence 임계값만 사용 → 오탐률 30%
개선 후
바운딩박스 형태·이동량 복합 조건 추가 → 오탐률 10% 이하, 실시간 25FPS+ 유지
CASE 02 · Pre/Auto-Labeling 구조적 유사 경험
경량 모델 + 프레임 샘플링으로 실시간 추론 파이프라인 성능 확보
전체 프레임 추론 시 속도 저하 → YOLOv5s 경량 모델 전환 → 프레임 샘플링 전략 적용 → 실시간 추론 25FPS+ 확보
모델 추론 결과를 품질 손실 없이 처리량을 확보해야 하는 문제를 경량 모델 적용과 샘플링 전략으로 해결한 경험입니다. Pre-labeling·Auto-labeling 파이프라인 구축 시 대량 데이터를 빠르게 처리하면서도 pseudo label 품질을 유지해야 하는 요구사항과 동일한 트레이드오프 설계 경험입니다.
CASE 03 · 데이터 파이프라인 운영 경험
대규모 수집 파이프라인의 무인 자동화 + 데이터 일관성 검증
반복 수작업 데이터 수집 → Selenium 자동 수집 설계 → Pandas 정제 + Docker/Cron 무인화 → 반복 수작업 100% 자동화
키노라이츠 콘텐츠 메타데이터 1,000건 이상을 평균 10분 내 수집하는 파이프라인을 설계하고, 수집 데이터의 정제·가공 로직과 일관성 검증을 직접 구현했습니다. Docker 컨테이너화와 Cron Job으로 일 1회 무인 자동 수집 체계를 구축한 경험은 사내 라벨링 파이프라인의 안정적 운영, 품질 모니터링 체계 수립에 바로 적용할 수 있습니다.
CASE 04 · 데이터 품질 점검 / 이상치 처리
대용량 공공데이터 결합 및 이상치 처리로 예측 모델 정밀도 개선
공공데이터 120만 건 + 기상데이터 결합 → 데이터 품질 점검·이상치 처리 → 시간대/요일/날씨별 피처 엔지니어링 → MAE 기준 오차 15%↑ 개선
서울시 따릉이 약 120만 건 데이터와 기상 데이터를 결합하는 과정에서 결측치·이상치를 점검·처리하고 피처 엔지니어링을 수행했습니다. 대용량 데이터셋의 품질을 구조적으로 관리하고 정량 지표(MAE)로 개선을 검증한 경험은 데이터셋 품질 분석 업무에 직접 적용 가능합니다.

Section 06
왜 로보티즈AI · 데이터 Curation인가
REASON 01
모델 출력을 정량적으로 검증해 본 경험이 있습니다
CCTV 낙상 감지 프로젝트에서 바운딩박스·Confidence Score를 정량 기준으로 분석해 오탐률을 30%에서 10% 이하로 낮췄습니다. 라벨 정확도·일관성 검증, 데이터셋 품질 분석은 제가 이미 깊게 다뤄본 문제의 연장선입니다.
REASON 02
반복 작업을 자동화 파이프라인으로 바꾼 경험이 있습니다
