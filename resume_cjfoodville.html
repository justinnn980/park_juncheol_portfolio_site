Section 02
문제를 어떻게 해결했나
PROBLEM 01
서비스 전환 시 사용자 이탈 최소화 — 기존 여정 분석 후 재설계
사용자 행동 데이터 분석 → 단계별 이탈 구간 식별 → 신규 앱 구조 재설계 → 전환 이탈 최소화
헬로tv 앱 → 오초이스 전환 시, 기존 사용자의 콘텐츠 탐색 → 시청 → 재시청 요청 흐름을 분석해 신규 앱 구조에 맞게 재설계. 서비스 전환 직후 이탈 최소화. 뚜레쥬르 앱에서 홈 탐색 → 상품 상세 → 장바구니 → 주문 → 재방문 흐름 최적화에 동일 방식 적용 가능.
PROBLEM 02
FCP 39% 단축 — 구매 전환에 직결되는 성능 지표 개선
FCP / LCP 지표 분석 → 장바구니 진입 전 병목 식별 → Lazy Loading · 스켈레톤 UI · CDN → FCP −39%, LCP −10.8%
초기 화면 로딩 지연이 탐색 이탈률을 높이는 문제를 FCP/LCP 지표로 정량화. 장바구니 진입 전 -53% 병목 구간을 데이터로 식별 후 Lazy Loading·스켈레톤 UI·CDN 적용으로 FCP 39% 단축. 뚜레쥬르 앱 홈 화면 로딩 → 상품 탐색 → 구매 전환율 개선에 동일 접근 적용 가능.
PROBLEM 03
멤버십 설계 — 연결 상태·기기·콘텐츠 단위 사용자 여정 분리
셋톱박스 연동 상태 분석 → 연결 / 기기 / 콘텐츠 단위 설계 → 멤버십 기능 완성
셋톱박스 최대 3대 페어링 멤버십에서 연결 상태, 기기 노출, 콘텐츠 접근 단위로 사용자 여정을 분리 설계해 상태 변화를 즉시 반영. 뚜레쥬르 앱의 회원 등급 / 쿠폰 적립·사용·만료 / 포인트 흐름과 구조적으로 동일한 패턴 — 직접 적용 가능.
PROBLEM 04
앱 재배포 없이 콘텐츠·이벤트 즉시 반영하는 운영 구조 구축
배포 주기 의존성 문제 식별 → 탐색 화면 WebView 분리 → 이벤트·프로모션 즉시 반영
탐색 화면을 앱 외부 웹으로 분리해 앱 재배포 없이 콘텐츠 구성·배너·화면 변경을 즉시 반영 가능한 운영 구조 구현. 뚜레쥬르 앱의 시즌 이벤트·신제품 프로모션·쿠폰 이벤트를 빠르게 적용하는 운영 효율화에 동일 원리 활용 가능.
PROBLEM 05
구독 서비스 커머스 구조 — 등록·갱신·해지·이력 흐름 설계
구독 흐름 분석 (등록/갱신/해지) → 도메인 분리 설계 → Soft Delete 이력 보존 → 통합 대시보드 구현
구독 등록·수정·해지·조회 흐름을 기준으로 User / Subscription / Community 도메인 분리 설계. Soft Delete로 구독 이력 데이터 보존. 자신의 구독 현황과 지출을 한눈에 확인하는 통합 대시보드 기획 및 구현. 뚜레쥬르 앱의 주문 이력·쿠폰 사용 내역 관리와 동일 구조.
