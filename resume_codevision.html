03Quality & Problem Solving

품질 관리 역량

실무에서 마주한 어노테이션 품질 문제와 일관성 유지 원칙을 정리했습니다.

품질 과제 오클루전 처리 일관성

가림(Occlusion)이 발생한 객체를 어디까지 박스로 잡을 것인지 기준이 모호했다

여러 객체가 겹치는 장면에서 가려진 객체의 경계를 어느 범위까지 박스·폴리곤으로 지정해야 하는지 가이드라인마다 기준이 달랐습니다. 임의로 판단하면 작업자마다 결과가 달라져 모델 학습 데이터 일관성이 무너집니다.

원칙가이드라인의 occluded 속성 정의를 최우선 기준으로 삼고, 애매한 경우 가이드라인 해석 범위 안에서 가장 보수적인 기준(추정 가능한 범위만 포함)을 적용했습니다. 기준이 불명확한 케이스는 QA 코멘트에 기록해 검수자와 확인하는 절차를 따랐습니다.

품질 과제 Object Tracking — ID 연속성

영상에서 객체가 화면 밖으로 나갔다 재진입할 때 같은 ID로 연결해야 하는지 판단이 필요했다

Object Tracking 작업에서 특정 프레임에 객체가 사라졌다가 다시 등장할 때, 동일 객체로 같은 track_id를 유지해야 하는지 새 ID를 부여해야 하는지 명확한 판단 기준이 필요했습니다. 잘못된 ID 처리는 모델의 ReID(Re-Identification) 학습 품질에 직접 영향을 줍니다.

원칙가이드라인의 이탈 프레임 기준(보통 3~5프레임)을 우선 적용하고, 재등장 위치와 외형 일치 여부를 동시에 판단했습니다. 경계 케이스는 보수적으로 신규 ID를 부여하고 QA 코멘트에 판단 근거를 기록하여 검수자가 일괄 확인할 수 있도록 했습니다.

품질 과제 개발자 관점 활용

컴퓨터공학 배경으로 AI 모델이 어떤 데이터 구조와 품질을 요구하는지 이해하고 작업에 적용했다

단순히 가이드라인만 따르는 것을 넘어, AI 모델의 입력 데이터 형식과 학습 원리를 이해하면 어떤 어노테이션이 모델 성능에 중요한지 판단할 수 있습니다. 컴퓨터공학 전공 + 모빌리티AI 복수전공 과정에서 객체 검출·세그멘테이션 모델의 데이터 파이프라인을 학문적으로 학습했습니다.
