03Technical Deep-dives

기술 과제와 LLM 연결

실무에서 해결한 기술 과제가 LLM 서비스 개발에 어떻게 연결되는지 정리했습니다.

과제 LLM API 통합 — API 클라이언트 설계

복잡한 외부 API를 앱에 안정적으로 통합하고, 인증 토큰 만료 시 자동 갱신 구조를 만들어야 했다

OTT 서비스 앱에서 콘텐츠 API·DRM 라이선스 서버·결제 서버를 동시에 호출했습니다. JWT Access Token이 만료되면 사용자 경험을 끊지 않고 자동으로 Refresh Token을 통해 갱신해야 했습니다. LLM API(OpenAI, Anthropic)를 앱·서버에 통합하는 구조와 완전히 동일합니다.

JWT 자동 갱신 흐름 = LLM API 키 관리 패턴
API 요청
↓
401 Unauthorized토큰 만료
↓ 자동 감지
Refresh Token 갱신인터셉터 레이어
↓ 새 토큰 발급
원래 요청 재시도 — 사용자 경험 유지

해결Dio 인터셉터 레이어에서 401 감지 → Refresh Token 갱신 → 원래 요청 재시도 흐름을 구현했습니다. 사용자 입장에선 로그인 세션이 끊기지 않고, 시스템은 항상 유효한 토큰을 보장합니다. LLM API Key 교체, Rate Limit 처리, Fallback 모델 전환에 동일한 패턴을 적용할 수 있습니다.

토큰 만료 자동 처리 — 사용자 경험 무중단 LLM API 클라이언트 설계에 직접 전이 Rate Limit / Fallback 패턴 동일 구조
과제 실시간 스트리밍 응답 — WebSocket 설계

서버의 처리 결과를 클라이언트에 실시간으로 전달하고, 충돌 없이 모든 사용자에게 동기화해야 했다

"숫자 연결" 경매 게임에서 여러 사용자가 동시에 숫자를 올리면, 서버의 최종값을 모든 클라이언트에 즉시 전달해야 했습니다. LLM 응답 스트리밍(Server-Sent Events, WebSocket)과 완전히 동일한 구조입니다. 클라이언트가 로컬 상태를 믿지 않고 서버 결과만 신뢰하는 설계가 핵심이었습니다.

WebSocket 실시간 동기화 = LLM 스트리밍 응답 패턴
WebSocket 경매 게임
사용자 입력 이벤트
↓
서버 단일 상태 처리
↓ broadcast
모든 클라이언트 즉시 갱신
LLM 스트리밍 응답
사용자 프롬프트 입력
↓
LLM 추론 중 (chunk 생성)
↓ stream
클라이언트 실시간 토큰 렌더링

해결서버에서 단일 상태를 관리하고 WebSocket으로 모든 클라이언트에 broadcast하는 구조를 설계했습니다. 클라이언트는 UI 반응만 담당하고 실제 데이터는 서버 결과 기준으로만 동기화합니다. LLM chunk 스트리밍에서도 동일하게: 서버에서 token을 생성할 때마다 SSE/WebSocket으로 전달하고 클라이언트는 append만 수행합니다.

실시간 동기화 구조 설계 LLM 스트리밍 응답 패턴과 동일 3일 기획~배포 End-to-End
과제 RAG 데이터 파이프라인 이해 — 어노테이션 경험

AI 모델이 원하는 데이터 품질·구조를 만드는 과정을 실무에서 직접 경험했다

5개 AI 데이터 전문 기업에서 BBox·Polygon·Segmentation·Keypoint·Object Tracking 어노테이션을 수행하며, AI 모델이 실제로 어떤 형식과 품질의 데이터를 필요로 하는지 직접 경험했습니다. RAG 시스템에서 문서를 청킹·임베딩하는 과정은 어노테이션 레이블 구조와 동일한 원리입니다.

어노테이션 데이터 구조 = RAG 문서 파이프라인 원리
원본 데이터이미지/영상/문서
→
구조화 레이블링어노테이션/청킹
→
임베딩/인덱싱벡터 저장
→
모델 학습/ 검색 활용

연결어노테이션 경험을 통해 비정형 데이터를 AI가 처리할 수 있는 구조로 변환하는 과정을 이해합니다. RAG 파이프라인에서 의약품 문서·임상 데이터를 청킹하고 임베딩하는 작업에 동일한 사고방식이 필요합니다. 데이터 품질이 모델 성능을 결정한다는 원칙을 실무에서 내면화했습니다.

AI 학습 데이터 구조 이해 RAG 청킹·임베딩 원리 연결 5개 기관 데이터 품질 관리 경험

구독 통합 관리 서비스 "내일(Dama)"을 2인 팀으로 기획부터 배포까지 완성했습니다. User·Subscription·Community 도메인을 분리 설계하고, JWT 인증·MySQL 데이터 계층·Docker 컨테이너 환경까지 전구간을 책임졌습니다. LLM 기반 서비스도 프롬프트·API 통합·서버·배포를 일관되게 설계할 수 있습니다.
