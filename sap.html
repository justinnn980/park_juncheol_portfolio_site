① Spring Boot 기반 SAP OData 연동 데모

개인 프로젝트 · Java 17, Spring Boot, JPA, PostgreSQL, Docker Compose, Node.js(Mock SAP)

GitHub

github.com/justinnn980/SAP

목표

외부 주문 시스템이 SAP OData API로 주문을 전송하는 인터페이스 시나리오를 백엔드로 구현. 단순 내부 함수 호출이 아닌, Mock SAP OData 서버를 별도 Node.js 프로세스로 분리하고 Spring Boot가 HTTP로 호출하는 실제 외부 연동 구조를 재현했습니다.

주요 구현

  • SAP OData API 연동 구조 : /sap/opu/odata/sap/Z_ORDER_SRV/Orders EntitySet POST 요청/응답 파싱을 직접 구현 — SAP OData URL 구조, EntitySet 형식 그대로 재현
  • SAP 전송 상태 관리 : 주문 생성 → SAP 전송 요청 → 응답 수신 → 상태 업데이트(성공/실패/대기) → 전송 로그 저장 흐름 전체 구현
  • 실패 로그 및 재처리 API : 요청/응답 payload 로그 저장, 실패 건 목록 조회, 재처리 트리거 API 구현 — 실무 인터페이스 개발의 핵심인 추적 가능성(Traceability)과 장애 대응 구조를 의식해 설계
  • 연동 모니터링 화면 : SAP 전송 현황 대시보드 — 성공/실패 건수 확인, 실패 로그 상세 조회, 재처리 실행 제공

어필 포인트

외부 시스템 → SAP OData 연동 패턴을 실서비스와 동일한 구조로 재현. 단순 호출에서 끝내지 않고 실패 로그와 재처리 흐름까지 구현하여 SAP 인터페이스 개발 실무 감각을 보여줍니다.

④ SAPUI5 매출 리포트 OData 앱

개인 프로젝트 · SAPUI5, XML View, Controller, ODataModel V4, manifest.json, Spring Boot (OData Mock 백엔드)

목표

Spring Boot OData Mock 서비스를 SAPUI5 ODataModel로 연결하는 XML View 기반 앱 구현. React 프론트엔드 경험을 SAP 표준 UI 개발 방식(MVC + ODataModel)으로 확장했습니다.

주요 구현

  • SAPUI5 표준 앱 구조 : Component.js, manifest.json에 OData 서비스 등록, XML View 선언형 화면 구성, Controller 기반 로직 분리
  • ODataModel 데이터 바인딩 : XML View의 테이블에 EntitySet 직접 바인딩, Controller에서 기간/고객/제품/청구상태 조건을 sap.ui.model.Filter 객체로 생성하여 ODataModel에 적용 → OData $filter 요청이 실제 백엔드로 전달되는 흐름 전체 구현
  • formatter 적용 : 금액 숫자 포맷, 청구 상태 코드→한글/색상 표시 formatter 구현

어필 포인트

manifest/View/Controller/ODataModel 구조를 코드로 직접 구현하여, React 경험을 SAP 표준 UI 개발 방식(MVC + ODataModel)으로 확장할 수 있음을 보여줍니다.
