EXPERIENCE 01
WebSocket 서버 중심 상태관리 = LLM 스트리밍 응답과 동일 구조
서버 단일 상태 관리 → WebSocket broadcast → 전체 클라이언트 실시간 동기화 → LLM 스트리밍과 동일 패턴
숫자 대결 게임에서 서버를 단일 진실 원천으로 삼고 WebSocket으로 모든 클라이언트에 실시간 상태를 브로드캐스트하는 구조를 3일 단독 구현. 이 구조는 sLLM이 토큰을 생성할 때마다 WebSocket으로 클라이언트에 스트리밍하는 구조와 완전히 동일. 서버 측 상태 관리·충돌 방지·클라이언트 동기화 경험을 LLM 스트리밍 서비스에 직접 적용 가능.

EXPERIENCE 04
복잡한 상태 분리 설계 — LLM 애플리케이션의 상태 관리에 직접 적용 가능
셋톱박스 3대 페어링 문제 → 연결/기기/콘텐츠 단위 분리 → Riverpod 상태 변화 즉시 반영
셋톱박스 최대 3대 페어링 멤버십의 연결 상태·기기 노출·콘텐츠 접근을 Riverpod StateNotifierProvider로 단위별 분리 관리. LLM 어플리케이션에서 요청 중(loading) / 스트리밍(streaming) / 완료(done) / 오류(error) 상태를 클린하게 분리하는 패턴과 동일 — 복잡한 AI 서비스 상태 관리에 직접 적용 가능.
EXPERIENCE 05
API 통합 (REST + WebSocket + JWT + HTTPS) — LLM API 클라이언트 개발 역량
JWT 인증 설계 → REST + WebSocket 통합 → AWS EC2 + HTTPS 보안 → 전구간 보안 통신 완성
이통사 방화벽 차단 문제를 AWS EC2 Elastic IP + HTTPS 전환으로 해결해 전구간 보안 통신을 구성. JWT Access/Refresh Token 인증 흐름을 Flutter에서 직접 구현. LLM API 클라이언트 개발에 필요한 인증·보안·프로토콜 통합 역량을 실서비스에서 검증.
