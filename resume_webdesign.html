OTT 앱 마이그레이션 — 웹 UI 전환 및 성능 최적화

2024.02 ~ 2025.10 (1년 10개월) · React 18, TypeScript, Flutter 3, AWS S3 · 팀 4명

목표

콘텐츠 포스터 UI 요소가 많은 영화 상세 화면을 React 기반 웹으로 전환하여, 운영팀이 앱 배포 없이 화면 콘텐츠를 실시간으로 수정·반영할 수 있는 구조를 구현했습니다. LG헬로비전·딜라이브 서비스를 "오초이스" 앱으로 마이그레이션하는 과정에서 담당했습니다.

주요 경험

  • 초기 로딩 체감 속도 개선: 스켈레톤 UI 적용으로 콘텐츠 로딩 중 빈 화면 노출 제거. Lazy Loading으로 불필요한 리소스 사전 로딩 차단 → FCP 39%, LCP 10.8% 개선
  • 포스터 이미지 자산 최적화: 이미지 리사이징 및 포맷 최적화로 1,000건 이상 이미지 관리. 페이지 렌더링 속도와 시각적 품질 동시 개선
  • 이관 유연성 확보: 콘텐츠 변경 시 앱 재배포 없이 웹 화면만 수정해 즉시 반영하는 구조 구성 → 프로모션/이벤트 등 이관 요구사항에 즉각 대응 가능
  • 브랜드 UI 일관성 유지: 디자이너와 협업해 컴포넌트 단위로 화면 품질 기준을 관리. 브랜드 아이덴티티를 화면에 구현한 경험 획득
  • Flutter WebView 연동: 앱 내 WebView로 React 웹 화면을 호출하는 구조로 앱·웹 콘텐츠 흐름 통합

성과

  • 브랜드 UI 일관성을 유지하면서 성능과 이관 효율성을 함께 개선
  • 디자이너, 이관팀과 협업하며 화면 기준을 브랜드 기준으로 구현하는 경험 획득
  • LG헬로비전(매출 1조), 딜라이브(매출 3,500억) 서비스 화면 실서비스 운영

  • 실시간 상태 UI 반영: WebSocket으로 수신한 서버 상태 변화를 Riverpod StateNotifier로 UI에 즉시 반영. 지연 없이 상태가 화면에 표시되도록 구성

성과
