문제 Android Native DRM API

NCG DRM에서 Multi DRM으로 전환하면서 Android 네이티브 라이선스 API를 직접 연동해야 했다

딜라이브 모바일TV에서 NCG DRM만 지원하던 플레이어를 Multi DRM(Widevine/PlayReady) 환경으로 전환해야 했습니다. 기존 플레이어 구조를 교체하면 재생·DRM 로직 전체를 다시 짜야 했기 때문에, Android Native DRM API(MediaDrm)를 MethodChannel로 브릿징하여 단일 플레이어 구조를 유지하는 방식으로 이전 복잡도를 최소화했습니다.

기존 재생 로직 재사용 가능플레이어 교체 없이 DRM만 전환

해결Android Native MediaDrm API를 MethodChannel로 Flutter 레이어에 노출하여, 플레이어 코드 변경 없이 DRM 방식만 교체했습니다. Widevine 라이선스 요청·갱신·해제 흐름을 생명주기와 결합하여 메모리 누수 없이 처리했습니다.

플레이어 재구성 없이 DRM 전환 완료 Android Native MediaDrm API 직접 연동 MethodChannel 실무 적용
문제 Android 생명주기 분석

외부 플레이어 라이브러리 버그로 오디오가 이중 출력됐고, 공식 패치를 기다릴 수 없었다

쇼츠형 콘텐츠를 스크롤하면 특정 조건에서 플레이어가 두 번 초기화되어 사운드가 겹쳐 출력됐습니다. 라이브러리 교체 또는 공식 패치를 기다리면 수주가 걸리는 상황이었고, 라이브러리 소스를 직접 분석해 Android Activity 생명주기 이벤트 처리의 버그 시점을 특정해야 했습니다.

해결외부 라이브러리를 fork하여 소스를 직접 분석했고, Android Activity 생명주기 이벤트에서 중복 초기화가 발생하는 시점을 특정하여 onCreate/onResume/onDestroy 기준으로 생성·재생·종료 시점을 명확히 분리했습니다. 당일 배포로 해결했습니다.

문제 Flutter 2→3 / Android 빌드

Flutter 2→3 메이저 업그레이드 시 Android 빌드 충돌과 의존 라이브러리 비호환 문제가 동시에 발생했다

딜라이브 모바일TV 마이그레이션 과정에서 Flutter 2 → Flutter 3 업그레이드가 필요했습니다. Gradle 버전 요구사항 변경, deprecated Android API, 플레이어 라이브러리 비호환이 한꺼번에 터졌고, 각 의존성의 Android 빌드 호환성을 하나씩 검증해야 했습니다.

Flutter 3 업그레이드 — Android 빌드 충돌 해결 흐름
Flutter 3 업그레이드 시도Gradle / AGP 버전 충돌 + deprecated API 오류
↓
Gradle / AGP 버전 매핑 분석flutter.gradle → AGP 호환 테이블 참조
↓
의존 라이브러리 Android 호환성 점검플레이어 / DRM / 카메라 순차 검증
↓
플레이어 재구성비호환 라이브러리 교체 + API 마이그레이션
↓
Android / iOS 동시 빌드 성공실서비스 배포 완료

해결Flutter/Gradle/AGP 버전 호환 테이블을 기준으로 단계별로 버전을 고정하고, 의존 라이브러리를 플레이어→DRM→기타 순으로 점검했습니다. deprecated Android API는 현행 대응 API로 교체하고, 비호환 플레이어 라이브러리는 직접 재구성했습니다.

Flutter 2→3 업그레이드 완료 Android 빌드 호환성 직접 검증 실서비스 배포 완료
문제 수치 기반 성능 최적화

WebView 전환 후 초기 로딩이 느려졌고, 주관적 판단이 아닌 수치로 병목을 특정해야 했다

콘텐츠 화면을 WebView로 교체하자 체감 로딩 속도가 저하됐습니다. 주관적 판단 대신 FCP(First Contentful Paint)/LCP(Largest Contentful Paint)를 측정하고 병목 구간을 정량적으로 특정한 뒤 개선했습니다.

↓

✓ AFTER

해결스켈레톤 UI로 FCP 체감 시간을 단축하고, Intersection Observer 기반 Lazy Loading으로 뷰포트 밖 이미지는 요청하지 않았습니다. 서버에서 화면 크기에 맞게 리사이징된 이미지를 제공하여 네트워크 대역폭과 디코딩 비용을 동시에 줄였습니다.
