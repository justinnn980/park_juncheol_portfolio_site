03Growth & Track Record

성장과정과 성공·실패 사례

실무에서 부딪힌 문제를 통해 다져온 일하는 방식이 SAP 인터페이스 개발에 어떻게 연결되는지 정리했습니다.

성향 근본 원인을 끝까지 추적하는 집요함

화면 레벨 조건 추가로는 해결되지 않아, 원본 플레이어 라이브러리를 직접 수정했다

쇼츠형 화면에서 플레이어가 두 번 초기화되어 사운드가 중복 출력되는 문제가 있었습니다. 표면적인 증상에 조건문을 추가하는 방식으로는 근본적으로 해결되지 않는다는 것을 확인하고, 원인을 Android 생명주기 레벨까지 추적했습니다.

해결원본 플레이어 라이브러리를 직접 수정해 Android 생명주기 레벨에서 단일 오디오 출력 구조로 재구성했습니다. 표면적 증상보다 원인을 추적하는 데 시간을 쓰는 편이며, 이 집요함은 영향 범위가 넓은 SAP 인터페이스 디버깅에서도 동일하게 발휘될 역량이라고 생각합니다.

근본 원인 추적 — 라이브러리 레벨 수정 SAP 인터페이스 디버깅에 동일하게 적용 가능
보완책 임시조치로 영향 최소화 후 근본원인 분석 시간 확보

집요함이 시간 지연으로 이어질 위험을, 임시조치와 근본원인 분석을 분리하는 습관으로 보완했다

근본 원인을 찾는 데 집중하면 예상보다 시간이 더 걸리고, 다른 업무 우선순위가 흐트러질 위험이 있었습니다. Proxy IP 변경으로 방화벽 차단 문제가 발생했을 때도 같은 위험이 있었습니다.

05Motivation & Vision

지원동기 및 비전

더존ICT그룹은 국내 기업들의 ERP와 디지털 전환을 가장 가까이에서 책임지고 있는 회사라고 생각합니다. (주)홈초이스에서 Spring Boot 기반 백엔드와 외부 시스템 연동, 운영 환경의 장애 대응을 직접 수행하며 "안정적으로 동작하는 시스템을 만드는 일"의 무게를 체감했습니다. 이 경험을 기반으로 SAP 영역에 도전하기 위해 자격증이나 이론 공부에 머물지 않고, Mock SAP 서버 분리·전송 상태 관리·재처리 API까지 갖춘 OData 연동 구조를 직접 구현하며 SAP 인터페이스 개발에서 중요한 추적 가능성과 장애 대응 구조를 의식했습니다.
더존ICT그룹에 입사한다면 이렇게 직접 만들어본 경험을 실제 SAP 프로젝트의 인터페이스/연동 개발에 빠르게 적용하고 싶습니다. 또한 Claude Code 같은 AI 도구를 기획·설계·구현 전 과정에서 적극적으로 활용해온 경험을 살려 ERP 개발/유지보수 생산성을 높이는 데에도 기여하고 싶습니다. 장기적으로는 SAP 인터페이스 개발자로서 고객사의 실제 업무 흐름을 깊이 이해하고, 더존ICT그룹이 추구하는 디지털 전환 과정에서 안정적인 연동 구조를 설계할 수 있는 사람으로 성장하는 것이 비전입니다.
LG헬로비전(가입자 343만 명), 딜라이브(매출 3,500억) 두 통신사 서비스를 하나의 앱으로 마이그레이션하며 DRM 방식 차이, 셋톱박스 페어링 상태관리, Proxy/SSL 인프라 문제를 직접 해결한 경험은 ERP처럼 여러 시스템과 데이터가 맞물려 있는 환경에서 원인을 좁혀가는 감각으로 이어집니다. Flutter 기반 모바일 개발과 Spring Boot 기반 백엔드 개발을 모두 실무에서 경험했기 때문에, 클라이언트와 서버, SAP와 같은 외부 시스템까지 연결되는 전체 흐름을 폭넓게 이해하고 협업할 수 있다는 점도 강점이라고 생각합니다.
