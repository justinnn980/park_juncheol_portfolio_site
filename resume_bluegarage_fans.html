Section 02
실서비스에서 해결한 문제들
PROBLEM 01 · 라이브 도메인 적합성
쇼츠형 화면 오디오 이중 출력 — 플레이어 생명주기 분리로 완전 해결
스크롤 시 이전 영상 오디오 잔존 → 라이브러리 소스 직접 분석 → 생성 / 재생 / 종료 생명주기 명시 분리 → 오디오 겹침 완전 해결
딜라이브 OTT 앱에서 쇼츠형 피드 스크롤 시 이전 영상 오디오가 겹쳐 출력되는 문제. API 문서만으로 해결 불가 — betterPlayer 라이브러리 소스를 직접 분석해 내부 생명주기 처리 방식을 파악. 영상 전환 시 플레이어 생성/재생/종료 단계를 명시적으로 분리 호출해 완전 해결. FANS 라이브 스트리밍·아티스트 콘텐츠 재생에서도 동일하게 적용 가능한 역량입니다.
PROBLEM 02 · 커뮤니티 도메인 적합성
셋톱박스 3대 실시간 상태 동기화 — 다중 상태를 단위별로 분리 관리
최대 3대 기기 상태 혼재·렌더링 오류 → 연결·기기·콘텐츠 단위 분리 설계 → StateNotifierProvider 구조화 → 상태 변화 즉시 반영 · 유지보수성 개선
셋톱박스 최대 3대 페어링 상태를 단일 Provider로 관리하면 상태 혼재 및 렌더링 오류 발생. 연결 상태 / 기기 노출 / 콘텐츠 접근 권한을 별도 StateNotifierProvider로 분리해 각 단위의 상태 변화가 독립적으로 즉시 반영되도록 설계. FANS 커뮤니티의 팔로우 상태, 좋아요, 실시간 알림 등 다중 상태 흐름 관리에 동일하게 적용 가능한 구조 경험입니다.
PROBLEM 03 · 성능 최적화 역량
WebView 전환 후 성능 저하 — FCP 39%, LCP 10.8% 수치 개선
앱 배포 의존 + 초기 로딩 지연 → FCP/LCP 기준 병목 분석 → Lazy Loading · Skeleton UI · CDN 적용 → FCP −39% · LCP −10.8%
LG헬로비전 헬로tv 앱(케이블TV 가입자 343만 명) 전환 프로젝트에서 콘텐츠 탐색 화면을 WebView로 분리 후 초기 로딩 체감 속도 저하 발생. FCP/LCP 지표를 기반으로 병목 구간을 식별하고 CDN 전환, 번들 최적화, Lazy Loading, Skeleton UI를 순차 적용. FCP 39%, LCP 10.8% 개선 달성. 수백만 글로벌 유저가 사용하는 FANS 앱의 성능 병목 파악·해결에 즉시 기여 가능합니다.
변경 전
앱 코드 수정 → 빌드 → 스토어 심사(1~3일) → 배포 → 콘텐츠 반영
변경 후
React 웹 배포만으로 즉시 반영 (앱 배포 없음) · FCP −39% 달성
PROBLEM 04 · 커머스 도메인 적합성
낙관적 업데이트 + 서버 실패 시 자동 롤백 — 사용자 혼란 없는 커머스 UX
삭제 요청 후 서버 응답 대기 중 UX 공백 → TanStack Query 낙관적 업데이트 → 서버 실패 시 이전 상태 자동 복원 → 반응성 개선 + 데이터 안정성 확보
OTT Admin(콘텐츠 관리 어드민)에서 영화/시리즈 삭제 시 서버 응답을 기다리는 동안 UI가 멈춰 보이는 UX 문제. TanStack Query의 onMutate 훅으로 삭제 즉시 UI 반영 후, 서버 실패 시 onError에서 이전 캐시 상태로 자동 롤백 구현. 커머스에서 카트 추가·수량 변경·쿠폰 적용 등 사용자 입력이 즉시 반영되어야 하는 상황에 동일하게 적용 가능합니다.
PROBLEM 05 · DRM / 미디어 통합
NCG DRM → Multi DRM 전환 — 단일 플레이어 인터페이스로 통합
서비스마다 다른 DRM 방식 → DRM 타입 추상화 레이어 설계 → 플레이어 커스터마이징 → 서비스 무관 동일 시청 흐름
딜라이브 앱(NCG DRM)과 헬로tv 앱(Multi DRM: Widevine/FairPlay)의 DRM 방식 차이로 통합이 어려운 문제. DRM 방식을 추상화한 단일 플레이어 인터페이스를 설계하여 내부에서 DRM 타입을 분기 처리. 별도 솔루션 도입 없이 추가 비용·운영 리스크 없이 해결. 글로벌 콘텐츠를 제공하는 FANS 플랫폼의 미디어 시스템 이해에 직접 활용 가능한 경험입니다.

왜 Blue Garage · FANS인가
REASON 01
라이브 / 미디어 경험이 직접 연결됩니다
홈초이스에서 VOD 재생, DRM 통합, 플레이어 생명주기 관리를 실서비스에서 직접 다뤘습니다. 아티스트가 직접 송출하는 라이브와 채팅 기능 개발은 제가 쌓아온 미디어 스트리밍 구조 이해를 가장 직접적으로 활용할 수 있는 도메인입니다.
REASON 02
글로벌 서비스의 복잡성을 해결하고 싶습니다
LG헬로비전(343만 가입자) 규모의 OTT 앱 성능 개선 경험을 갖고 있습니다. 수백만 글로벌 팬이 사용하는 FANS 플랫폼에서 렌더링 병목 파악, 응답성 개선, 다국어 i18n 구조 설계를 함께 해결해나가고 싶습니다.
REASON 03
크로스플랫폼 개발자로서 즉시 기여합니다
Flutter와 React Native는 동일한 선언형 UI·상태관리·네이티브 브릿지 패러다임을 공유합니다. React 19 + TypeScript 실서비스 경험까지 갖추어, 학습 비용 없이 React Native 코드베이스에 즉시 기여할 수 있습니다.
