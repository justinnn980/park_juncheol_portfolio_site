내마(Dama) — 구독 정보 통합 관리 서비스

사용자의 구독 정보를 통합 관리하고, 구독 기반 커뮤니티 및 데이터 분석 확장을 고려하여 설계된 서비스. 단순 CRUD가 아닌 도메인 기반 설계(DDD)와 확장 가능한 구조를 목표로 개발했습니다.

Kotlin Spring Boot Spring Security JWT QueryDSL JPA PostgreSQL Flutter Docker
기간 및 참여인원
2026.01 ~ 2026.02 (2개월) / 개발자 2명, 디자이너 1명
  • Refresh Token Rotation 구현 — 탈취된 토큰으로 재사용 공격 가능성 차단
  • 이전 토큰 해시를 DB에 저장(prevTokenHash), 동일 토큰 재사용 감지 시 즉시 탈취 처리. Refresh Token을 HMAC-SHA256 해시한 후 저장, ThreadLocal로 Mac 인스턴스 재사용해 객체 생성 비용 절감
  • 구독 비용 분석 기능 — 전 구독 비용과 지난달 대비 증가분을 한 번에 조회하는 API 구현
  • 지난달 말일 기준 활성 구독 스냅샷 조회 후 이번 달과 비교, 절약/지출 메시지 생성. 결제일 D-3 이내 구독에 대해 "N일 뒤 결제" 알림 메시지 자동 생성
  • 보안 설계 — 타 사용자의 구독 접근 시 403 대신 404 반환으로 존재 여부 노출 차단
  • 모든 엔티티에 Soft Delete 정책과 BaseEntity 구조 적용, User / Subscription 컨텍스트 분리

Study Tracker API — 학습 Todo 관리 REST API

JWT 인증, Redis 캐싱, CI/CD 배포까지 실무 흐름을 적용한 Spring Boot 기반 Todo 관리 API. 단순 CRUD가 아닌 인증, 사용자 데이터 격리, 캐싱, 중복 요청 방지, 배포까지 실무 흐름 전체를 구현했습니다.

Java 21 Spring Boot Spring Security JWT Spring Data JPA PostgreSQL Redis Docker GitHub Actions Render
기간 및 참여인원
2025 / 개인
Swagger 문서
study-tracker-api-7stm.onrender.com
  • JWT 직접 구현 — jjwt 등 외부 라이브러리 없이 HMAC-SHA256 서명 직접 구현
  • javax.crypto.Mac으로 서명 생성, MessageDigest.isEqual()로 타이밍 어택 방어까지 고려한 서명 검증 구현. JwtAuthenticationFilter로 모든 보호 API 요청에서 토큰 검증, todoId + userId 복합 조건으로 타 사용자 리소스 접근 차단
  • Redis 기반 성능 최적화 — 통계 조회 API의 반복 쿼리 부하 문제 해결
  • @Cacheable / @CacheEvict로 통계 캐시(TTL 60초) 적용, 생성/수정/완료/삭제 시 해당 사용자 캐시 즉시 무효화. 조회 패턴 분석 후 복합 인덱스 2개 설계 (user_id,id / user_id,completed,id)
  • 멱등성 키 기반 중복 요청 방지 — 네트워크 재시도 시 Todo 중복 생성 문제 해결
  • IdempotencyStore 인터페이스로 추상화, 운영에는 Redis, 테스트에는 InMemory 구현체로 교체 가능한 구조 설계. Idempotency-Key를 Redis에 5분 TTL로 저장, 중복 요청 시 409 Conflict 응답으로 차단
  • 서비스 레이어 CQRS 분리 — UserService(쓰기) / UserQueryService(읽기) 분리로 책임 명확화
  • CI/CD 파이프라인 구축 — GitHub Actions main push 시 자동 테스트/빌드, Docker 멀티 스테이지 빌드 + Render 배포. Actuator health endpoint로 운영 상태 확인, 단위/통합 테스트 60건 작성 (MockMvc, Mockito, JUnit 5)
