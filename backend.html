OTT 앱 마이그레이션 — 다중 SO 인증 API 구축

2024.02 ~ 2025.10 (1년 10개월) · Java 17, Spring Boot 2, MySQL, DynamoDB, AWS · 팀 4명

목표

홈초이스tv와 오초이스 앱으로의 마이그레이션을 위해 LGH로비전, 딜라이브, HCN 등 사업자마다 다른 인증 서버 구조를 단일 API로 처리하고, 콘텐츠 배포 파이프라인을 자동화했습니다.

주요 경험

  • 다중 SO 회원 인증 API: SO별 SiteInfo로 암호화 방식(AES-256-ECB / AES-128-CBC), 인증 URL, 키를 분리 관리. 앱 → PVS → SO 인증 서버 → 오초이스 회원 등록 흐름을 API 버전별(v1/v2/v3)로 설계
  • AES 암호화 통신 구현: 외부 SO 서버와 주고받는 요청/응답에 평문 전송 불가 조건 충족. 요청 수신 시 복호화, 처리 후 응답 암호화를 각 엔드포인트에 적용
  • Quartz 스케줄러 배치 자동화: 5분 주기 Quartz 잡 6개(배포, 인코딩, 배포상태, 업로드, 벌크파일) 구성. AWS S3 연동 미디어 업로드, Ateme 인코딩 시스템 연동
  • Proxy 서버 IP 고정: 로드밸런싱 환경에서 방화벽 차단 문제 해결 — EC2 기반 Proxy에 Elastic IP 할당하여 고정 IP 구조로 재구성
  • HTTPS 전환: Proxy 서버 SSL 미적용으로 인한 통신 실패 해결 — 와일드카드 SSL 인증서 적용으로 전체 구간 HTTPS 구성

성과

  • 서로 다른 인증 구조의 3개 SO사를 단일 API 아키텍처로 통합
  • 콘텐츠 업로드부터 배포까지 수동 작업 전 과정을 Quartz 배치로 자동화
  • 방화벽·SSL 인프라 이슈를 직접 분석하고 구조적으로 해결

내마(Dama) — 구독 정보 통합 관리 서비스

2026.01 ~ 2026.02 (2개월) · Kotlin, Spring Boot, JWT, QueryDSL, PostgreSQL, Docker · 팀 3명

주요 경험

  • Refresh Token Rotation 구현: 이전 토큰 해시를 DB에 저장(prevTokenHash), 동일 토큰 재사용 감지 시 즉시 탈취 처리. Refresh Token을 HMAC-SHA256 해시 후 저장, ThreadLocal로 Mac 인스턴스 재사용해 객체 생성 비용 절감
  • 구독 비용 분석 API: 지난달 말일 기준 활성 구독 스냅샷 조회 후 이번 달과 비교, 절약/지출 메시지 자동 생성. 결제일 D-3 이내 구독에 "N일 뒤 결제" 알림 자동 생성
  • 보안 설계: 타 사용자의 구독 접근 시 403 대신 404 반환으로 리소스 존재 여부 노출 차단
  • DDD 설계: User / Subscription 컨텍스트 분리, 모든 엔티티에 Soft Delete + BaseEntity 구조 적용

성과

  • 토큰 탈취·재사용 공격을 RTR 패턴으로 구조적으로 차단
  • 도메인 기반 분리 설계로 커뮤니티·데이터 분석 기능 확장 가능한 구조 확보

Study Tracker API — 학습 Todo 관리 REST API

2025 · Java 21, Spring Boot, Redis, PostgreSQL, Docker, GitHub Actions, Render · 개인

Swagger 문서

study-tracker-api-7stm.onrender.com/swagger-ui

주요 경험

  • JWT 직접 구현: jjwt 없이 javax.crypto.Mac으로 HMAC-SHA256 서명 생성. MessageDigest.isEqual()로 타이밍 어택 방어까지 고려한 서명 검증. todoId + userId 복합 조건으로 타 사용자 리소스 접근 차단
  • Redis 캐싱 최적화: @Cacheable / @CacheEvict로 통계 캐시(TTL 60초) 적용. 생성/수정/완료/삭제 시 해당 사용자 캐시 즉시 무효화. 복합 인덱스 2개 설계 (user_id,id / user_id,completed,id)
  • 멱등성 키 기반 중복 요청 방지: IdempotencyStore 인터페이스로 추상화, 운영에는 Redis, 테스트에는 InMemory 구현체로 교체. Idempotency-Key를 5분 TTL로 저장, 중복 시 409 Conflict 응답
  • 서비스 레이어 CQRS: UserService(쓰기) / UserQueryService(읽기) 분리로 책임 명확화
  • CI/CD 파이프라인: GitHub Actions main push 시 자동 테스트/빌드, Docker 멀티 스테이지 빌드 + Render 배포. 단위/통합 테스트 60건 (MockMvc, Mockito, JUnit 5)

성과

  • 외부 라이브러리 없이 JWT 보안 원리를 직접 구현하며 암호화 이해도 심화
  • 인터페이스 기반 추상화로 테스트/운영 환경 전환이 자유로운 구조 설계
  • 테스트 60건, CI/CD 파이프라인까지 갖춘 실무급 REST API 완성
