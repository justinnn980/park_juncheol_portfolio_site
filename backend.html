오초이스 OTT 모바일 앱 개발

2024.02 ~ 2025.10 (1년 10개월) · Flutter, Dart, Riverpod, WebView, AWS EC2 · 팀 4명

목표

기존 파트너사 앱 사용자들이 오초이스 앱으로 자연스럽게 전환할 수 있도록, 서비스 이용 흐름을 유지하면서 앱을 교체하는 것이 핵심 과제였습니다.

문제 해결

  • 영상 이중 재생 문제: 쇼츠형 화면에서 영상 넘길 때 이전 영상 소리 중복 재생 발생. 플레이어 라이브러리 동작 방식 직접 확인 후, 플레이어 생성·재생·종료 시점을 명확히 분리하여 해결
  • 영상 저작권 보호 방식 통합: 딜라이브(NCG 방식) · 오초이스(Multi 방식) 보호 체계가 달라 재생 구조 통일 불가 → 플레이어를 직접 수정해 보호 방식이 달라도 동일한 화면·흐름으로 영상 재생
  • Flutter 2 → 3 업그레이드: 영상 플레이어·일부 라이브러리 호환 문제 발생 → 비호환 항목 하나씩 확인·수정하며 기능 이상 없이 업그레이드 완료
  • 앱 배포 없는 콘텐츠 운영 구조: 콘텐츠 탐색 화면을 웹 기반으로 분리 후 WebView로 호출하는 구조로 변경 → 이후 콘텐츠·배너 변경 시 앱 심사 없이 즉시 반영
  • 초기 로딩 속도 개선: 웹 전환 후 첫 화면 로딩 지연 문제 → 불필요 파일 제거, CDN 이미지, 화면에 보이는 부분 우선 로딩, Skeleton UI 적용 → FCP 39% 단축, LCP 10.8% 개선
  • 서버 IP 변경으로 인한 방화벽 차단: 중계 서버 IP 변경 시 보안 정책에 의해 요청 차단 → 중계 서버에 고정 IP 할당. 이후 HTTPS 전환 시 SSL 인증서 적용으로 전체 구간 암호화

성과

  • LG헬로비전(매출 1조, 343만 가입자) · 딜라이브(매출 3,500억) 두 차례 대형 앱 전환 완료
  • 앱 배포 없이 콘텐츠를 운영할 수 있는 구조 구축 → 이관 요구사항 즉각 대응 가능
  • FCP 39% 개선, 영상 보호 방식 단일화, 서버 통신 안정화 동시 달성

Study Tracker API — 학습 Todo 관리 REST API

2025 ~ 2026.06 · Java 21, Spring Boot, Redis, PostgreSQL, Docker, GitHub Actions, Render · 개인

링크

Swagger 문서  ·  GitHub

문제 해결

  • 타 사용자 데이터 접근 차단: Todo 번호만 알면 다른 사용자 항목에 접근 가능한 보안 문제 → 조회·수정·삭제 시 항상 현재 로그인 사용자 정보까지 함께 확인하도록 처리
  • 통계 조회 성능 개선: 반복 통계 조회 시 매번 DB 계산 → Redis에 60초 임시 저장, Todo 변경 시 해당 사용자 캐시 즉시 무효화. 복합 인덱스 2개 추가 설계로 조회 속도 개선
  • 중복 생성 요청 방지: 버튼 이중 클릭·네트워크 재전송으로 Todo 중복 생성 가능 → 요청마다 고유 멱등성 키 부여, Redis에 5분 보관, 동일 키 재요청 시 차단(409)
  • 배포 후 서버 상태 확인: 상태 확인 API 구성 + 로그/환경변수/DB/Redis 연결 점검 항목 정리한 운영 문서 작성

주요 구현

  • JWT 직접 구현: jjwt 없이 javax.crypto.Mac으로 HMAC-SHA256 서명 생성. MessageDigest.isEqual()로 타이밍 어택 방어
  • 멱등성 추상화: IdempotencyStore 인터페이스로 추상화 → 운영은 Redis, 테스트는 InMemory 구현체로 교체 가능
  • CQRS: UserService(쓰기) / UserQueryService(읽기) 분리
  • CI/CD: GitHub Actions → Docker 멀티 스테이지 빌드 → Render 자동 배포. 단위/통합 테스트 60건 (MockMvc, Mockito, JUnit 5)

담아(Dama) — 구독 정보 통합 관리 서비스

2026.01 ~ 2026.02 (2개월) · Kotlin, Spring Boot, JWT, QueryDSL, PostgreSQL, Docker · 팀 3명

주요 경험

  • Refresh Token Rotation — 토큰 탈취 방어: 로그인 유지 토큰을 암호화하여 DB 저장, 이미 사용한 토큰으로 재접근 시 해당 계정의 모든 세션 즉시 강제 종료. HMAC-SHA256 해시 후 저장, ThreadLocal로 Mac 인스턴스 재사용해 객체 생성 비용 절감
  • 구독 비용 분석 API: 지난달 말일 기준 활성 구독 스냅샷 조회 후 이번 달과 비교, 절약/추가 지출 메시지 자동 생성. 결제일 D-3 이내 구독에 "N일 뒤 결제" 알림 자동 생성
  • 보안 설계: 타 사용자 구독 접근 시 403 대신 404 반환 → 데이터 존재 여부 자체를 외부에 노출하지 않도록 처리
  • DDD 설계: User / Subscription 컨텍스트 분리, 모든 엔티티에 Soft Delete + BaseEntity 구조 적용

성과

  • 토큰 탈취·재사용 공격을 RTR 패턴으로 구조적으로 차단
  • 도메인 기반 분리 설계로 커뮤니티·데이터 분석 기능 확장 가능한 구조 확보
