02Problem Solving

문제 해결 — 다날 업무와의 연결

실무에서 마주한 문제와 해결 과정을 데이터 흐름으로 풀어냈습니다. 다날 금융 서비스와 직결되는 경험들입니다.

문제 다날 필수 — 보안통신 · 결제 안전성

Elastic IP 고정 후 Proxy 방화벽이 차단돼 앱 전체가 API를 호출할 수 없었다

AWS EC2 Elastic IP를 고정한 뒤 일부 이동통신사 네트워크에서 Proxy 방화벽이 해당 IP를 차단하기 시작했습니다. 기존 HTTP 통신은 중간 Proxy에 노출되어 차단 대상이 됐고, 앱 전체의 API 호출이 불가능해졌습니다. 결제 서비스에서 동일한 문제가 발생하면 결제 자체가 불가능해지는 치명적 상황입니다.

해결SSL 인증서를 EC2 인스턴스에 적용하고 전체 API 엔드포인트를 HTTPS로 전환했습니다. TLS 암호화로 Proxy가 내용을 검사할 수 없게 되어 차단이 해제됐습니다. 결제 서비스에서의 카드 정보·인증 토큰 전송 보호와 완전히 동일한 구조입니다.

전체 API 통신 차단 문제 해결 TLS 암호화로 데이터 보안 확보 결제 정보 보호와 동일 구조 적용
문제 다날 필수 — 결제 토큰 검증 흐름

NCG DRM → Multi DRM 전환 시, 라이선스 검증 흐름이 플레이어마다 달라 재생이 불안정했다

기존 NCG DRM에서 Multi DRM(Widevine/FairPlay)으로 전환하면서 각 플레이어가 라이선스 서버에 인증을 요청하는 흐름이 달라졌습니다. 라이선스 검증 로직이 플레이어 코드 곳곳에 흩어지면 토큰 갱신 타이밍이 어긋나 재생이 끊기거나 인증이 실패합니다. 결제 서버 토큰 검증/갱신 흐름과 구조가 동일합니다.

DRM 라이선스 검증 = 결제 토큰 검증 동일 패턴

해결플레이어 커스터마이징으로 DRM 라이선스 검증 로직을 단일 레이어에 통합했습니다. 모든 재생 요청이 이 레이어를 반드시 통과하도록 만들어 토큰 갱신 타이밍이 일치하고 단일 재생 구조가 유지됐습니다. 결제 서비스의 PG 토큰 검증·갱신 로직과 동일하게 적용할 수 있습니다.

라이선스 검증 단일 레이어 통합 재생 불안정 완전 해소 결제 토큰 검증 패턴에 직접 전이 가능
문제 다날 필수 — React + Flutter 연동

콘텐츠 선택·구매 흐름이 React WebView와 Flutter Native 사이를 오가야 했다

콘텐츠 화면을 React + TypeScript WebView로 분리한 후, 사용자가 콘텐츠를 선택하거나 구매 버튼을 누르면 Flutter Native에서 재생·결제 흐름이 이어져야 했습니다. 두 환경의 데이터 동기화 없이는 화면이 멈추거나 상태가 어긋납니다. 다날 금융 서비스 앱의 결제 WebView ↔ Native 연동과 동일한 구조입니다.

문제 다날 우대 — 대규모 B2C 성능 최적화

전국 케이블TV 가입자가 앱 진입 시 첫 화면 로딩이 느려 이탈률이 높았다

전국 규모 B2C 서비스(LG헬로비전, 가입자 수백만)에서 앱 첫 화면 로딩 지연이 이탈률에 직결됐습니다. FCP(First Contentful Paint)와 LCP(Largest Contentful Paint) 지표를 측정했을 때 로딩 병목이 확인됐고, 이를 데이터 기반으로 개선했습니다.

문제 Flutter 2→3 마이그레이션 · 안정성

실서비스 마이그레이션 중 Flutter 2→3 업그레이드 시 의존 라이브러리 호환성이 무너졌다

드라이브 모바일TV 앱을 스카이이어스로 마이그레이션하는 도중 Flutter 3 업그레이드가 필요했습니다. Null Safety 도입으로 기존 DRM 플레이어 라이브러리가 호환되지 않았고, Android Activity 생명주기 레벨의 플레이어 동작 방식도 변경됐습니다.

해결의존 라이브러리의 호환성을 사전 전수 점검하고, Android 생명주기 레벨에서 플레이어 생성/재생/종료 시점을 명확히 분리했습니다. 라이브러리 소스를 직접 분석해 플레이어 중복 실행 문제를 완전히 해소했습니다. 실서비스에 영향 없이 마이그레이션을 완료했습니다.

실서비스 중단 없이 Flutter 3 전환 완료 플레이어 중복 실행 완전 해소 라이브러리 소스 직접 분석 역량
