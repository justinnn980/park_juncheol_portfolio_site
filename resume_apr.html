문제 APR 우대 — Riverpod / BLE 인접

최대 3개 디바이스를 관리하며, 연결 상태 변경 시 앱 전체 화면이 실시간으로 바뀌어야 했다

사용자가 셋톱박스(디바이스)를 연결/변경/종료하면 홈/재생/설정 등 모든 화면이 즉시 갱신되어야 했습니다. AGE-R 앱의 BLE 디바이스 연결 상태관리와 동일한 패턴으로, Riverpod 기반 단일 상태 관리 지점을 설계했습니다.

문제 APR 필수 — 웹/JS 연동

콘텐츠 선택/재생/구매 흐름이 WebView(JS)와 Native(Flutter) 사이를 오가야 했다

콘텐츠 화면을 React/TypeScript WebView로 분리한 후, 사용자 인터랙션이 발생하면 Native Flutter 레이어에서 재생/결제 흐름이 이어져야 했습니다. 두 환경의 데이터 동기화 구조 없이는 화면 동작이 멈추거나 상태가 어긋납니다.

문제 APR 필수 — 수치 기반 성능 최적화

WebView 전환 후 초기 로딩이 느려졌고, 주관적 판단이 아닌 수치로 병목을 특정해야 했다

콘텐츠 화면을 WebView로 교체하자 체감 로딩 속도가 저하됐습니다. APR이 강조하는 "논리적인 근거와 수치 기반 커뮤니케이션"을 실천하여, FCP(First Contentful Paint)/LCP(Largest Contentful Paint)를 측정하고 병목 구간을 정량적으로 특정했습니다.

문제 코드/엔진 레벨 디버깅

외부 플레이어 라이브러리 버그로 쇼츠 화면에서 오디오가 이중 출력됐고, 공식 수정을 기다릴 수 없었다

쇼츠형 콘텐츠를 스크롤하면 특정 조건에서 플레이어가 두 번 초기화되어 사운드가 겹쳐 출력됐습니다. APR이 요구하는 "코드/엔진/OS 레벨까지 깊이 있는 문제 파악"이 필요한 케이스였습니다. 라이브러리 교체나 공식 패치를 기다리면 수주가 걸리는 상황이었습니다.

문제 APR 우대 — 레거시 분석 역량

이종 레거시 앱 2개의 화면/재생/DRM 구조를 분석하고 단일 앱으로 안정적으로 전환해야 했다

LG헬로비전(매출 1조) "헬로tv"와 딜라이브(매출 3,500억) "딜라이브 모바일TV"를 오초이스 앱으로 통합했습니다. 각 앱마다 화면 진입 경로/재생 구조/DRM 방식이 달랐고, 사용자가 기존 경험을 유지해야 한다는 조건이 있었습니다.
