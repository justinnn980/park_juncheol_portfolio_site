문제 APR 우대 — Riverpod 상태관리 셋톱박스 페어링 상태관리

하나의 앱에서 최대 3개 디바이스를 관리하며, 연결 상태에 따라 콘텐츠가 실시간으로 바뀌어야 했다

사용자가 셋톱박스(디바이스)를 연결·변경·종료하면 앱 화면 전체가 즉시 갱신되어야 했습니다. 최대 3개 디바이스를 동시에 관리해야 하고, 홈·재생·설정 등 여러 화면에서 같은 상태를 참조해야 하는 복잡한 구조였습니다. AGE-R 앱처럼 디바이스와 연동되는 앱에서 요구되는 실시간 상태관리와 동일한 문제입니다.

문제 APR 필수 — 웹/JS 연동 WebView ↔ Native 연동 구조 설계

콘텐츠 선택·재생·구매 흐름이 WebView(JS)와 Native(Flutter) 사이를 오가야 했다

콘텐츠 화면을 WebView(React/TypeScript)로 분리한 뒤, 사용자가 콘텐츠를 선택하거나 구매 버튼을 누르면 Native(Flutter)에서 재생·결제 흐름이 이어져야 했습니다. 두 환경 사이에 데이터를 주고받는 구조 없이는 화면 동작이 멈추거나 상태가 어긋납니다.

문제 APR 우대 — 레거시 분석 역량 이종 서비스 간 앱 마이그레이션

레거시 앱 2종의 화면·재생·DRM 구조를 분석하고 오초이스 앱으로 안정적으로 전환해야 했다

LG헬로비전(매출 1조)의 헬로tv 앱, 딜라이브(매출 3,500억)의 딜라이브 모바일TV 앱을 오초이스 앱으로 통합해야 했습니다. 각 앱마다 화면 흐름, 재생 구조, DRM 방식이 달랐고, 사용자 경험은 유지해야 한다는 조건이 있었습니다.

해결 각 앱의 화면 진입 경로, 콘텐츠 재생 흐름, DRM 방식을 먼저 문서화하고 오초이스 구조와 매핑했습니다. 단순 기능 이전이 아닌 사용자가 기존에 익숙했던 UX 패턴을 유지하면서 내부 구조만 교체했습니다. 이 역량은 복잡한 레거시 코드베이스를 파악하고 개선하는 업무에 바로 적용됩니다.

2건 마이그레이션 완수 기존 사용자 UX 유지 레거시 구조 분석 역량 검증
문제 이종 DRM 호환 · 플레이어 커스터마이징

NCG DRM과 Multi DRM의 방식 차이로 재생이 되지 않았고, 플레이어를 직접 수정해야 했다

딜라이브는 NCG DRM, 오초이스는 Multi DRM(Widevine·FairPlay)을 사용하고 있어 단일 플레이어로는 두 콘텐츠를 재생할 수 없었습니다. 별도 솔루션 도입 시 라이선스 비용과 외부 의존성이 추가되는 문제도 있었습니다.
