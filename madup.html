배치 파이프라인 & 이벤트 기반 인증 API

2024.02 ~ 2025.10 (1년 10개월) · Java 17, Spring Boot, Quartz, DynamoDB, AWS · 팀 4명

매드업 Prism과의 접점

콘텐츠 수집·인코딩·배포를 자동화한 Quartz 배치 파이프라인은 Prism의 광고 성과 데이터 수집·적재 ETL과 동일한 문제 구조입니다. 파편화된 외부 시스템 데이터를 자동으로 통합·처리하는 경험을 실무에서 쌓았습니다.

주요 경험

  • Quartz 배치 파이프라인: 5분 주기 잡 6개(배포, 인코딩, 배포상태, 업로드, 벌크파일) 구성. AWS S3 연동 미디어 업로드, Ateme 인코딩 시스템 연동으로 콘텐츠 전 과정 자동화
  • 다중 SO 인증 API: 3개 사업자의 서로 다른 인증 구조(암호화 방식, URL, 키)를 SO별 SiteInfo 설정으로 분리 관리. v1/v2/v3 API 버전으로 각 사업자 요구사항 대응
  • AES 암호화 통신: AES-256-ECB / AES-128-CBC를 각 엔드포인트에 적용. 요청 수신 시 복호화, 응답 시 암호화로 전체 통신 구간 보안 처리
  • 인프라 문제 자주적 해결: 로드밸런싱 Proxy IP 차단 문제 → Elastic IP 도입. HTTPS 통신 실패 → 와일드카드 SSL 직접 적용

성과

  • 수동 처리하던 콘텐츠 배포 파이프라인 전 과정을 배치로 자동화
  • 외부 시스템 연동 이슈(방화벽, SSL)를 보고 없이 스스로 분석·해결

실시간 이벤트 처리 — WebSocket 브로드캐스트

2026.01 ~ 진행 중 · Spring Boot, STOMP WebSocket, Flutter, Render · 개인

매드업 EDA와의 접점

Kafka/SQS 기반 이벤트 드리븐 아키텍처와 동일한 원칙 — "명령과 상태 전파를 분리" — 을 STOMP WebSocket으로 직접 구현했습니다. REST는 트리거, 상태 변화는 이벤트 브로드캐스트로 처리하는 구조입니다.

주요 경험

  • 이벤트 기반 아키텍처 설계: 클라이언트 중심 업데이트는 동시성 제어가 어렵다고 판단, 서버를 단일 상태 관리 지점으로 설정. REST API는 변경 명령(trigger) 역할만 담당
  • STOMP 브로드캐스트: SimpMessagingTemplate.convertAndSend("/topic/total", ...)로 변경 결과를 전체 구독자에게 즉시 전파. Polling 없이 실시간 동기화
  • 클라이언트 연동: 앱 시작 시 REST로 현재 상태 초기화, 이후 STOMP WebSocket 구독으로 전환. 5초 자동 재연결 처리
  • 기획~배포 3일 단독 완성: 프론트엔드, 백엔드, Render 배포 전 과정을 AI 도구 활용으로 빠르게 진행

Study Tracker API — Redis 캐싱 & CI/CD

2025 · Java 21, Spring Boot, Redis, PostgreSQL, Docker, GitHub Actions, Render · 개인

Swagger 문서

study-tracker-api-7stm.onrender.com/swagger-ui

주요 경험

  • JWT 직접 구현: 외부 라이브러리 없이 HMAC-SHA256 서명 구현. MessageDigest.isEqual()로 타이밍 어택 방어. todoId + userId 복합 조건으로 타 사용자 리소스 접근 차단
  • Redis 캐싱 최적화: @Cacheable/@CacheEvict로 통계 캐시(TTL 60초). 조회 패턴 분석 후 복합 인덱스 2개 설계 (user_id,id / user_id,completed,id)
  • 멱등성 키 중복 방지: IdempotencyStore 인터페이스로 추상화, 운영은 Redis / 테스트는 InMemory로 교체 가능한 구조. 5분 TTL, 중복 시 409 Conflict
  • CI/CD 파이프라인: GitHub Actions main push 시 자동 테스트/빌드, Docker 멀티 스테이지 + Render 배포. 단위/통합 테스트 60건 (MockMvc, Mockito, JUnit 5)
